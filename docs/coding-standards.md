# Coding Standards

## Reglas para v0.1

1. **Server Component por defecto.** `'use client'` solo cuando no queda otra.
2. **Un componente por archivo.** Export nombrado.
3. **Props tipadas** con `type`. Mismo archivo.
4. **`any` prohibido.** Usa `unknown` si toca.
5. **Nombres:** PascalCase (componentes), camelCase (funciones), kebab-case (archivos).
6. **Sin comentarios** que digan lo obvio. Comenta el "por qué", no el "qué".
7. **Sin abstracciones prematuras.** Si algo se usa una vez, no lo refactorices.

Esta semana la prioridad es enviar. No te detengas en perfección de código.
